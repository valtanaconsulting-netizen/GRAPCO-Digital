BIM ↔ GRAPCO Produc-App

Plan formal para integrar el modelo BIM (Building Information Modeling) con la plataforma GRAPCO Produc-App. Vincula el modelo 3D con el control de productividad real, habilitando vistas 4D (tiempo) y 5D (costo) sincronizadas con los datos del campo.

1 ¿Por qué incorporar BIM?

BIM es la representación digital tridimensional con metadata (costo, tiempo, materiales) de la obra. La plataforma GRAPCO Produc-App ya gestiona el control operativo. Al vincular ambos, cada punto del modelo 3D conoce su estado real de avance, costo y productividad — convirtiendo planos estáticos en dashboards vivos.

🎯

Visualizar productividad en contexto físico

En lugar de leer una tabla "CONCRETO ARMADO Eje A: CPI 0.78", ver el muro real coloreado en rojo en el modelo 3D. Comprensión instantánea para gerencia y cliente.

⏱️

Simulación 4D de cronograma

Ver cómo crece la obra semana a semana en una animación. Útil para planificación, comunicación con el cliente y validación del Lookahead a 6 semanas.

💰

Costos 5D vinculados a elementos

Cada elemento del modelo (columna, muro, losa) carga sus HH ejecutadas y costos reales en S/. Trazabilidad completa desde el plano hasta la planilla.

2 Arquitectura propuesta

El modelo BIM no se incrusta dentro de la plataforma. Se conecta vía API con un visor BIM externo (Autodesk APS o IfcOpenShell). La plataforma actúa como capa de datos que enriquece el modelo con información operativa.

3 Capacidades 4D y 5D

Una vez vinculados, la plataforma habilita 4 capacidades nuevas que hoy requieren software especializado adicional como Navisworks o Synchro 4D.

🎬 4D · Simulación temporal

Animación de la obra en el tiempo. Cada semana muestra qué elementos se ejecutaron en color verde, qué quedó pendiente en rojo.

Caso de uso: presentación al cliente cada mes mostrando el avance proyectado vs real con replay del Lookahead.

Disponibilidad: Fase 4 del roadmap VDC

💰 5D · Vinculación con costo

Cada elemento muestra al hacer click: HH ejecutadas, costo MO real (HE 60/100 incluido), comparación contra presupuesto.

Caso de uso: identificar visualmente las zonas más costosas de la obra y atacar problemas focalizados.

Disponibilidad: Fase 4 del roadmap VDC

🚧 Restricciones georreferenciadas

Las restricciones del Last Planner System se asocian a elementos BIM. El residente ve en 3D dónde están los problemas que están bloqueando la ejecución.

Caso de uso: sesión ICE semanal navegando el modelo 3D en lugar de hablar de "esa zona del nivel 2".

Disponibilidad: Fase 4 + integración LPS

📷 Foto vs modelo

Comparativa lado a lado: foto real del avance subida desde el celular del capataz vs el modelo BIM teórico.

Caso de uso: auditoría de calidad y detección temprana de desviaciones constructivas.

Requiere módulo de fotos (Bloque 8 sugerido)

5 Roadmap de implementación

Implementación incremental en 4 fases de 3 a 6 semanas cada una. Cada fase entrega valor independiente y no depende de las siguientes.

FASE 1 · 3 SEMANAS

Vinculación WBS ↔ Elementos BIM

Modificar el modelo de datos: agregar campo bimGuids: [string] a cada actividad de la WBS. Crear interfaz manual para asociar elementos del modelo a actividades.

Entregable: WBS extendida con metadata BIM. Sin visor todavía, sólo persistencia de los IDs.

FASE 2 · 4 SEMANAS

Visor BIM básico embebido

Integrar Autodesk Forge Viewer (gratuito hasta cierto volumen) como iframe dentro de una nueva pestaña "🏗️ Modelo 3D" en el Dashboard Ingeniero. Carga del archivo .ifc o .rvt convertido.

Entregable: Visor 3D navegable. Click en elemento → muestra los datos GRAPCO asociados.

FASE 3 · 4 SEMANAS

Coloreado por estado / CPI

Pintar los elementos del modelo según indicadores: verde = ejecutado, rojo = atrasado, amarillo = en proceso. CPI por elemento como heatmap. Filtros por partida, semana, capataz.

Entregable: Modelo BIM como dashboard visual. Reuniones ICE con el modelo de referencia.

FASE 4 · 6 SEMANAS

Simulación 4D + restricciones georreferenciadas

Animación temporal del avance. Restricciones del LPS pintadas en el modelo. Comparativa foto real vs BIM teórico (requiere módulo de fotos del Bloque 8).

Entregable: VDC completo Stanford CIFE. Caso de estudio cerrado para tesis.

8 Riesgos y mitigaciones

Implementar BIM en una organización que viene de planos 2D tiene riesgos predecibles.

Riesgo Probabilidad Mitigación
Modelo BIM desactualizado vs obra real Alta Definir responsable de modelo (Of. Técnica). Política de actualización semanal previa a sesión ICE.
Capataces no usan el visor 3D Media Visor sólo en Dashboard Ingeniero. Capataces siguen usando interfaz simple. BIM es para residente y oficina técnica.
Modelos demasiado pesados (lentos) Media Subdividir modelo por niveles/zonas. Forge Viewer maneja LOD (level of detail) automático.
Vinculación WBS ↔ BIM manual = error humano Alta Validación cruzada: si elemento no tiene actividad asociada, alerta automática. Auditoría mensual.
Cliente no tiene equipo para visualizar BIM Baja Forge Viewer corre en cualquier navegador. No requiere instalación. Click + JavaScript = listo.

9 Caso de uso · Sesión ICE semanal con BIM

Cómo cambia una sesión ICE viernes 4pm cuando se incorpora el visor BIM.

❌ ANTES — sin BIM

  • Residente proyecta tabla Excel con 50 filas de actividades.
  • "En la zona del eje C tenemos problema con el encofrado..." — capataz pregunta "¿en qué nivel?"
  • Discusión 15 min sobre QUÉ está siendo el problema.
  • "Mejor lo vemos mañana en obra" → reunión sin decisiones.

✅ DESPUÉS — con BIM integrado

  • Residente proyecta visor 3D con elementos coloreados según CPI.
  • "Esta zona naranja tiene CPI 0.65" → click → ve HH ejecutadas, capataz, fecha.
  • Decisión inmediata: "Asignar 2 personas más a esta zona la próxima semana".
  • Restricción asociada al elemento: queda georreferenciada para sesión siguiente.

10 FAQ implementación

¿Necesitamos comprar Revit?

No para visualizar. Sí para que la oficina técnica modele. Si ya tienes Revit (o el cliente entrega el modelo), basta. Alternativa libre: BlenderBIM + IfcOpenShell.

¿Funciona en celular?

Forge Viewer funciona en navegador móvil pero la experiencia es limitada (modelos pesados = lentos). Recomendado uso desde laptop/desktop. Capataz seguirá usando la interfaz móvil ligera.

¿Cuánto tarda convertir un modelo Revit a algo navegable?

Forge Viewer convierte automáticamente en 5-30 minutos según tamaño. Una vez convertido, queda en cache cloud. Solo se reconvierte si el modelo cambia.

¿Qué pasa si el cliente cambia el modelo a mitad de obra?

Los IDs de elementos suelen mantenerse en Revit (vía GUID). Si el cliente reemplaza un elemento, el sistema detecta GUID nuevo y alerta para revinculación manual. Validación cruzada lo previene.

¿BIM 4D no requiere software como Synchro o Navisworks?

Tradicionalmente sí. Con esta arquitectura, las "fechas" vienen del LPS de la plataforma y la "geometría" del modelo BIM. La animación 4D la generamos en JavaScript en el frontend. No requiere licencias adicionales.

¿Conviene esperar a tener BIM antes de implementar el LPS?

No. El LPS funciona perfectamente sin BIM (es lo que tenemos hoy). BIM es una capa adicional que multiplica el valor pero no es prerequisito. Madurar primero el LPS, luego sumar BIM.

¿Qué tamaño máximo de modelo soporta?

Forge Viewer maneja modelos de hasta 1 GB sin problemas. Para modelos más grandes (proyectos industriales tipo PTAR completas), subdividir por sectores. Cargar sólo el sector activo en cada sesión ICE.
