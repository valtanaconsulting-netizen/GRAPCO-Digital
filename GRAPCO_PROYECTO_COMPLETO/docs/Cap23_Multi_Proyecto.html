📚 CAPÍTULO 23 · BLOQUE 23

Multi-Proyecto Multi-Frente · La transformación SaaS

De plataforma de un solo proyecto a sistema enterprise multi-tenant. Jerarquía Empresa → Proyectos → Frentes → Actividades. RO independiente por nivel.

23.1 La transformación arquitectónica

Hasta el Bloque 22, GRAPCO Produc-App funcionaba como una aplicación de un solo proyecto. Si la empresa quería gestionar múltiples obras, necesitaba múltiples instancias de la app. El Bloque 23 transforma esto en una plataforma multi-proyecto multi-frente nivel enterprise, similar a Procore, Autodesk Construction Cloud o RIB CCS.

📢 El cambio de paradigma

Una empresa como GRAPCO con 5 obras simultáneas (PTARI Lima, Hospital Huarmey, Carretera Cusco, etc.) y cada obra con sus 3-4 frentes simultáneos, ahora gestiona TODO desde una sola plataforma. El Director General puede ver el dashboard de portfolio cada mañana y saber exactamente qué obra está perdiendo plata, dónde y por qué.

23.2 Jerarquía de datos en 4 niveles

EMPRESA (futuro multi-tenant)
└── PROYECTOS (PTARI, Hospital Huarmey, Carretera Cusco...)
    ├── codigo, nombre, cliente, ubicación
    ├── presupuesto, plazo, margen meta
    ├── estado, color
    └── FRENTES (F-NORTE, F-SUR, F-CENTRAL...)
        ├── codigo, nombre, responsable
        ├── presupuestoFrente, color, orden
        └── ACTIVIDADES WBS
            └── (cada actividad pertenece a UN frente de UN proyecto)
🏗️

Proyecto

Una obra completa. Tiene cliente, ubicación geográfica, contrato, plazo, presupuesto, estado.

📍

Frente

Una zona de trabajo dentro del proyecto. Tiene responsable, presupuesto propio, color visual.

📋

Actividad

Tarea WBS específica. Pertenece a UN frente de UN proyecto. Lleva proyectoId + frenteId.

23.4 Wizard de creación de proyecto

El wizard guía al usuario por 5 pasos secuenciales con validación en cada paso. Crea el proyecto y sus frentes iniciales en una sola transacción atómica usando writeBatch.

1️⃣

Identificación

Código único, nombre, cliente, tipo de obra (6 tipos preset), color del proyecto

2️⃣

Ubicación

Ciudad, región, dirección, coordenadas lat/lng (para mapa portfolio futuro)

3️⃣

Contrato

Presupuesto, moneda, fechas, plazo (auto-calculado), margen meta, estado

4️⃣

Frentes iniciales

Define los frentes en una sola pantalla. Mínimo 1, sin máximo. Cada uno con código, nombre, responsable, color

5️⃣

Confirmación

Revisión completa antes de crear. Cards de resumen por sección. Click final → todo en transacción atómica

23.5 Comparativo entre frentes

El motor del Bloque 21 (calcularROMensual) se aprovecha aquí filtrando los datos por frenteId. Cada frente obtiene su propio:

  • BAC, EV, AC (Earned Value Management completo)
  • CPI, SPI con coloreado de severidad
  • % avance físico
  • Margen real con coloreado relativo a la meta del proyecto
  • EAC (proyección al cierre)

El comparativo presenta tres vistas:

  1. Tabla comparativa con totales del proyecto en el footer
  2. Barras de avance visualmente comparables entre frentes
  3. Ranking de rentabilidad con medallas (🥇🥈🥉) según margen real

23.6 Estrategia de migración A1

Para no romper los datos existentes (Plan Maestro, Tareos, Kardex, etc. del Bloque 22), se implementó migración suave:

  1. Al primer arranque, SelectorProyectoFrente detecta que no hay proyectos creados
  2. Llama a asegurarProyectoDefault() que crea idempotentemente:
    • Proyecto default-ptari (PROYECTO PTARI Default)
    • Frente frente-general
  3. Las vistas legacy (que aún no tienen proyectoId) se asumen del proyecto default
  4. El usuario puede luego renombrar, agregar frentes, o crear proyectos adicionales
✅ Cero pérdida de información. Todo lo que ya existe (LPS, Tareos, Calidad, RO) sigue funcionando. Sólo se añade el contexto de proyecto/frente.

23.9 Para tu sustentación — el cierre maestro

📢 El argumento final

"GRAPCO Produc-App ya no es una plataforma para una obra, es una plataforma multi-proyecto multi-frente nivel SaaS empresarial. Cada empresa puede gestionar simultáneamente todas sus obras (PTARI Lima, Hospital Huarmey, Carretera Cusco, etc.), cada una con sus frentes simultáneos. El sistema calcula RO, CPI y SPI por frente, por proyecto, y consolidado por empresa. Esto NO existe en el mercado peruano integrado de esta forma. Las grandes empresas pagan SAP o Primavera por funcionalidades parciales; aquí está todo unificado y hecho para el contexto peruano (S/., RO mensual estilo Cosapi, APUs estilo S10)."
