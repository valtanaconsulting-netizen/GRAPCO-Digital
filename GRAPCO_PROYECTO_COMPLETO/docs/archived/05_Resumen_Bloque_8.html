Pipeline de subida

PasoQué pasa
1. CapturaInput capture="environment" abre la cámara trasera del celular directamente.
2. CompresiónCanvas reduce a max 1280px ancho, JPEG calidad 0.75. Una foto típica de 3MB queda en ~280KB.
3. SubidaFirebase Storage en registros/{actividadId}/{timestamp}_{i}.jpg. Progress % visible.

3 Detalle: 🏗️ Módulo BIM (Fases 1+2)

Implementación de las dos primeras fases del roadmap BIM definido en el documento estratégico previo.

Fase 1: Vinculación WBS ↔ Modelo BIM

  • Colección Firestore BIM_Vinculos: cada documento liga una actividad de la WBS con N GUIDs del modelo.
  • Modal de vinculación CRUD: selectores cascada Partida→Subpartida→Actividad + input de GUIDs uno por uno con validación.
  • Validación de GUIDs: acepta Revit GUID (UUID), IFC GlobalId (base64-like 22 chars con $), custom IDs alfanuméricos. Rechaza vacíos/espacios.
  • Tabla con filtros: por partida, estado (vinculado/sin vincular), búsqueda libre por GUID o actividad.
  • KPIs en cabecera: cobertura % (con color semáforo), GUIDs totales, promedio GUIDs/actividad, meta benchmark 80%.

Fase 2: Visor 3D embebido

  • 3 tipos de visor soportados: Autodesk Forge/APS, Speckle (open source), iframe URL personalizada.
  • Configuración por URN/URL: input para pegar el identificador del modelo. Click en "📍 Cargar" embebe el visor.
  • Iframe sandbox: allow-scripts allow-same-origin allow-popups. Acepta fullscreen.
  • Placeholder cuando no hay visor: pantalla con instrucciones de configuración + 4 cards de roadmap (Fase 1 lista · Fase 2 en config · Próximo paso · Capas siguientes).
  • Sin claves API hardcodeadas: la URN/URL la pone el usuario en runtime, queda en estado del componente (no persistida por seguridad).
🎯 Cómo activarlo en producción:
  1. Subir tu modelo Revit/IFC a Autodesk Construction Cloud (APS) — gratis hasta 50K cargas/mes.
  2. Obtener el URN del modelo desde la consola de APS (formato base64).
  3. En GRAPCO: ir a ANÁLISIS → Modelo BIM → Fase 2 → Visor 3D.
  4. Pegar URN → "📍 Cargar" → modelo navegable embebido.
  5. Alternativa open: Speckle.systems con free tier generoso, basta pegar la URL del stream.

4 Detalle: 📥 Exportación PDF con jsPDF

Enhancement sobre window.print() del Bloque 7. PDFs profesionales descargables.

Stack técnico

ComponenteDetalle
Carga dinámicajsPDF + jspdf-autotable se cargan desde CDN cloudflare la primera vez que se llama. No requiere npm install.
2 funciones expuestasexportarPDF({titulo, headers, rows, ...}) para datos + exportarTablaDOMaPDF({tablaSelector, ...}) para escraping de DOM existente.
Diseño del PDFHeader navy 22mm + acento dorado · "GRAPCO SAC" arriba izq · Título centrado · Fecha+hora arriba der · autoTable con headers navy y filas alternadas · footer con paginación.
OrientaciónConfigurable. Default landscape para tablas LPS anchas.
Botón en VDCCada vista LPS muestra dos botones: 🖨️ Imprimir (window.print) + 📥 Descargar PDF (jsPDF). Estado "⏳ Generando..." durante carga.
✅ Comparativa Imprimir vs PDF: Imprimir usa el navegador (depende del usuario), genera output variable. PDF nativo genera siempre el mismo archivo profesional, ideal para enviar al cliente o adjuntar al informe semanal.

6 Detalle: 📱 Responsive móvil + 🔍 Filtros alertas

Dos mejoras de UX que afectan toda la app.

Mobile responsive

  • Breakpoint isMobile: cambiado de 640px a 768px (cubre tablets en portrait y celulares en landscape).
  • Evento orientationchange: agregado para detectar rotación del dispositivo.
  • isTablet nuevo: variable adicional para 768-1024px.
  • CSS global responsive: overflow-x: hidden, -webkit-text-size-adjust: 100% (evita texto gigante iOS).
  • Touch targets ≥44px: norma iOS Human Interface Guidelines aplicada a todos los botones.
  • Inputs font-size 16px: evita el zoom automático de Safari iOS al hacer focus.
  • Botones con feedback táctil: :active aplica scale(0.97).

8 Para desplegar este bloque

Reemplazar archivos uno a uno en tu repo.

📋 Pasos:
  1. Reemplazar firebaseConfig.js con el contenido de firebaseConfig_NEW.js (agrega Storage).
  2. Crear estos archivos nuevos:
    • src/components/FotoUploader.jsx
    • src/utils/pdfExport.js
    • src/views/ingeniero/BIM.jsx
  3. Reemplazar archivos existentes: VDC.jsx, Capataz.jsx, Ingeniero.jsx, helpers.js, AlertasPanel.jsx, App.jsx, Auditoria.jsx, index.css.
  4. En Firebase Console → Storage: habilitar el servicio (gratis hasta 5GB) y crear bucket.
  5. Reglas de Storage temporales: allow read, write: if true (después endurecer junto con Firestore — sigue pendiente).
  6. npm run dev y probar.
