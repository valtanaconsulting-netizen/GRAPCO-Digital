24.2 Wizard de Plan Maestro con plantillas

24.2.1 Plantillas implementadas

Cada plantilla está basada en estructuras WBS reales del mercado peruano:

PlantillaActividadesDur. típicaCaso de uso
🏢 Edificación~50 partidas12 mesesEdificio multifamiliar/oficinas. Provisionales, estructuras, arquitectura, IIEE, IIS.
💧 Hidráulica~30 partidas10 mesesPTAR/PTAP. Civiles, líneas, equipamiento electromecánico, instrumentación.
🛣️ Vial~25 partidas14 mesesCarretera asfaltada. Movimiento de tierras, pavimentos, obras de arte, señalización.
🏭 Industrial~20 partidas8 mesesNave industrial. Civiles, estructura metálica, cobertura, IIEE industriales.
⚪ Vacía5 raíz—Solo crea las 5 partidas raíz. El usuario agrega el detalle.

24.2.2 Flujo del wizard (4 pasos)

  1. Plantilla: tarjetas visuales con icono, color, descripción y total de actividades.
  2. Configurar: frente destino, fecha de inicio (las actividades se distribuyen automáticamente en el cronograma), escala del presupuesto (slider 0.25× a 3×), y modo "reemplazar existentes".
  3. Vista previa: tabla con TODAS las actividades a crear, totales (presupuesto, HH, partidas hoja).
  4. Ejecutar: confirmación final. Crea las actividades en lotes de 400 con writeBatch de Firestore.
✅ Performance: Crear 50 actividades toma ~1 segundo en Firestore. Crear 200 actividades toma ~3 segundos.

24.3 Editor Masivo (CRUD estilo Excel)

24.3.1 Características

  • Tabla editable inline — todos los campos son editables directamente
  • Coloreado de cambios — filas modificadas en amarillo, eliminadas en rojo, nuevas en verde
  • Filtro por frente — vista solo del frente activo o todos
  • Cálculo automático — subtotal = metrado × P.U. en tiempo real
  • Selección múltiple para eliminación con checkbox
  • Agregar filas nuevas sin modal, directamente en la tabla
  • Guardado en lotes — todas las edits/inserts/deletes en transacciones atómicas
  • Botones contextuales — solo aparecen "Guardar" y "Descartar" cuando hay cambios pendientes

24.3.2 Algoritmo de guardado

// Pseudocódigo del guardarTodo()
batches = []
batch = writeBatch()
ops = 0

// 1. Updates
for (id, cambios) in edits:
  if id in eliminadas: continue
  batch.update(doc(id), { ...cambios, actualizadoEn: now })
  if ++ops >= 400: flush()

// 2. Inserts
for n in nuevas:
  if !n.codigo || !n.descripcion: continue
  batch.set(doc(), { ...n, creadoEn: now })
  if ++ops >= 400: flush()

// 3. Deletes
for id in eliminadas:
  batch.delete(doc(id))
  if ++ops >= 400: flush()

flush()
await Promise.all(batches.map(b => b.commit()))

24.5 Portfolio Dashboard ejecutivo

24.5.1 KPIs consolidados de la empresa

El dashboard del Director General muestra:

  • Cartera total — suma de BAC de todos los proyectos
  • Vendido acumulado — suma de EV (Earned Value) de todos
  • Costo real total — suma de AC de todos los proyectos
  • Margen global — (Vendido - Costo Real) / Vendido × 100
  • Avance global — EV / BAC × 100
  • CPI global — EV / AC consolidado

24.5.2 Ranking de rentabilidad

Los proyectos se ordenan por margen real con medallas 🥇🥈🥉. La tabla muestra para cada proyecto:

  • Código + nombre + ciudad + estado
  • BAC, EV, AC en S/.
  • % avance físico
  • CPI (con coloreado de severidad)
  • Margen real (con coloreado vs meta)
  • Cantidad de frentes y actividades
  • Botón "📍 Activar" para cambiar contexto

24.5.3 Alertas globales

El sistema detecta automáticamente:

  • CRÍTICO: proyectos con margen NEGATIVO (perdiendo plata)
  • ALTO: proyectos con margen < 50% de la meta
  • ALTO: proyectos con CPI < 0.85 (sobrecosto crítico)
  • ALTO: proyectos con SPI < 0.85 (atraso crítico)

Cada alerta tiene un botón "Investigar →" que activa ese proyecto y permite hacer drill-down.
