22.2 Panel Ejecutivo de Gerencia

22.2.1 Justificación

En empresas grandes de construcción, el Director de Operaciones necesita una vista que cruce todas las dimensiones del proyecto en una sola pantalla. No quiere abrir 6 sistemas distintos. Esta es la vista WOW que demuestra el valor del sistema integrado.

22.2.2 Componentes

  1. 4 KPIs financieros principales — Margen real, CPI, SPI, EAC al cierre
  2. Avance físico global — % avance + barra de progreso
  3. EVM completo — BAC, PV, EV, AC, VAC con interpretación
  4. 5 pilares laterales — Calidad, LPS, Materiales, OT, Tareos
  5. Alertas críticas — NCs críticas, margen negativo, CPI/SPI bajos, partidas en pérdida
  6. Partidas estrella — Las que aportan más al margen

22.2.3 Cálculos automáticos

El panel ejecuta en tiempo real:

// PPC (Last Planner System)
PPC = compromisos cumplidos / compromisos totales × 100

// % liberación calidad
% Liberación = protocolos liberados / total protocolos × 100

// Detección de alertas críticas
if (margenReal < 0)         → ALERTA CRÍTICA FINANCIERO
if (CPI < 0.85)             → ALERTA ALTO COSTO
if (SPI < 0.85)             → ALERTA ALTO CRONOGRAMA
if (NCs criticas > 0)       → ALERTA CRÍTICO CALIDAD

22.4 PETs vinculados al WBS

22.4.1 Innovación: PET ↔ WBS

Tradicionalmente, los PETs son documentos genéricos en PDF. Esta plataforma los vincula explícitamente a códigos WBS. Esto permite:

  • Saber qué actividades requieren cada PET
  • Bloquear el avance si el operario no firmó el PET correspondiente (futuro)
  • Auditoría: ¿quién firmó qué PET y cuándo?
  • Control de versiones del PET

22.4.2 Estructura del PET

{
  codigo: "PET-005",
  titulo: "Vaciado de columnas con bomba estacionaria",
  version: 2,
  alcance: "...",
  riesgosClave: "Caída a desnivel, golpes, atrapamientos",
  epp: "Casco, lentes, guantes, arnes con doble linea de vida",
  secuenciaTrabajo: "1. Inspeccionar EPP\n2. Verificar encofrado...",
  aplicaActividadesWBS: ["02.01.003", "02.01.004"],
  firmasOperarios: [
    { uid, nombre, email, fecha }
  ],
  estado: "vigente"
}

22.4.3 Flujo de firma digital

  1. Calidad/Ingeniero/Admin crea el PET con su contenido y selecciona las actividades WBS aplicables
  2. Capataz/Subcontratista ve los PETs de las actividades que va a ejecutar
  3. Antes de iniciar, hace click en "Firmar" declarando que leyó y entendió
  4. El sistema registra firmasOperarios[] con su uid, email, nombre y timestamp
  5. Auditoría disponible: cualquiera puede ver "Firmado por X, el dia Y"
