📋 Panorama del bloque

El panel de administración aparece como un módulo más en la barra superior, pero solo visible para usuarios con rol admin. Por debajo, está protegido por <RoleGuard> y por las reglas Firestore — un usuario no admin no podría acceder ni aunque manipule la URL.

2 Gestión de Usuarios

CRUD completo. Las operaciones críticas (crear, cambiar rol, eliminar) van por Cloud Functions con verificación de rol server-side — no se puede falsear desde el cliente.

Operaciones disponibles

  • Listar — tiempo real con onSnapshot, búsqueda por nombre/email, filtro por rol
  • Crear — modal con email, nombre, contraseña (con generador automático), rol
  • Cambiar rol — modal con 3 opciones visuales (capataz/ingeniero/admin)
  • Desactivar/Reactivar — soft delete (activo: false). Al desactivar, también revoca tokens (deslogueo inmediato)
  • Eliminar permanentemente — hard delete: borra cuenta Auth + perfil Firestore

Protecciones contra errores graves

  • No te puedes desactivar a ti mismo (lo previene Cloud Function)
  • No te puedes eliminar a ti mismo
  • No puedes quitarte el rol admin a ti mismo
  • Confirmación explícita antes de eliminar (texto requiere "OK")
  • Cada operación queda registrada en Auditoria_Seguridad

3 Visor de Auditoría

Lectura del log Auditoria_Seguridad con filtros y vista expandible por evento.

Filtros disponibles

  • Búsqueda libre — texto en acción, UID o detalles
  • Filtro por acción — dropdown con todas las acciones únicas detectadas
  • Filtro por UID parcial — útil para investigar a un usuario específico

Carga eficiente

Carga los últimos 150 eventos en orden cronológico inverso (más reciente arriba). Cada evento es expandible para ver el JSON completo con detalles, userAgent y fuente (cliente vs admin_panel).

Verificación server-side de rol admin

// El cliente envía Authorization: Bearer <idToken>
// El backend lo verifica y carga el perfil:

const requireAdmin = async (req) => {
  const idToken = req.get('Authorization').slice('Bearer '.length);
  const decoded = await admin.auth().verifyIdToken(idToken);
  const perfil = await admin.firestore().doc(`Usuarios/${decoded.uid}`).get();
  if (perfil.data().rol !== 'admin' || perfil.data().activo === false) {
    throw new Error('Acceso denegado: solo admins');
  }
  return { uid: decoded.uid };
};
🔒 Defensa en profundidad: Aunque las reglas Firestore ya validan, las Cloud Functions también validan. Si las reglas tuvieran un bug, las funciones siguen seguras (y viceversa). Capas redundantes.

🚀 Cómo desplegar este bloque

Archivos nuevos a crear

Archivos modificados

src/App.jsx                           // + import AdminPanel + módulo 'admin'
functions/index.js                    // + exports de admin.js

Pasos en orden

  1. Crea la carpeta src/views/admin/ y copia los 6 archivos JSX
  2. Copia src/utils/adminClient.js
  3. Reemplaza src/App.jsx con la versión del Bloque 13
  4. En el backend: copia functions_admin.js a functions/admin.js
  5. Edita functions/index.js y agrega al final: 
    const adminFns = require('./admin');
exports.adminCrearUsuario = adminFns.adminCrearUsuario;
exports.adminCambiarRol = adminFns.adminCambiarRol;
exports.adminDesactivarUsuario = adminFns.adminDesactivarUsuario;
exports.adminEliminarUsuario = adminFns.adminEliminarUsuario;
  6. Despliega backend: firebase deploy --only functions
  7. Build frontend: npm run build
  8. Deploy frontend: firebase deploy --only hosting
  9. Login como admin y verifica que aparece la pestaña 🛡️ Administración
  10. Prueba cada sub-vista: Resumen, Usuarios, Auditoría, Configuración, Salud
