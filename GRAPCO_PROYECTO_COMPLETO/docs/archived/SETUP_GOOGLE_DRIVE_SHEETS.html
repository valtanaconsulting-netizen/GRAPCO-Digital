GUÍA DE CONFIGURACIÓN

Setup · Google Drive + Sheets

Conecta la Cloud Function protocoloPdfFirmadoSync con tu Google Drive y tu Sheet de control para cerrar el workflow del CAL-FOR-006.

PROYECTO FIREBASE: grapco-demo-2026 TIEMPO TOTAL: ~15 min FRECUENCIA: Una sola vez

⚡ TL;DR — LO QUE VAS A HACER

1) Crear una Service Account en GCP (un "usuario robot" para la Cloud Function).

2) Habilitar Drive API y Sheets API con 1 click cada una.

3) Compartir tu carpeta de Drive y tu Sheet con el email de esa SA.

4) Activar plan Blaze + Budget Alert de $1/mes (red de seguridad — costo real esperado: $0).

5) Guardar 3 secrets en Firebase y desplegar la función. Listo.

1 Crear la Service Account

La SA es un "usuario robot" que la Cloud Function usará para escribir en Drive y Sheets en tu nombre — sin compartir tu cuenta personal.

1.1 · Abre Google Cloud Console

👉 console.cloud.google.com/iam-admin/serviceaccounts?project=grapco-demo-2026

Asegúrate de estar en el proyecto grapco-demo-2026 (lo ves arriba a la izquierda).

1.2 · Click en + CREATE SERVICE ACCOUNT

Llena los campos así:

  • Service account name: protocolos-bot
  • Service account ID: se autogenera
  • Description: Bot para archivado de protocolos en Drive y Sheets

Click CREATE AND CONTINUE → DONE. NO le des ningún rol IAM — no hace falta para Drive ni Sheets.

1.3 · Descargar la JSON key

Click sobre la SA recién creada → pestaña KEYS → ADD KEY → Create new key → tipo JSON → CREATE.

Se descarga un archivo grapco-demo-2026-XXXXX.json.

⚠️ Guárdalo bien — esta key solo se descarga una vez. Si la pierdes, hay que generar otra.

1.4 · Copia el email de la SA

Lo vas a usar 2 veces en los pasos 3 y 4. Tiene este formato:

protocolos-bot@grapco-demo-2026.iam.gserviceaccount.com

2 Habilitar Drive API + Sheets API

Las APIs vienen apagadas por defecto. Las prendes una vez y listo.

2.1 · Drive API

👉 console.cloud.google.com/apis/library/drive.googleapis.com

Click ENABLE.

2.2 · Sheets API

👉 console.cloud.google.com/apis/library/sheets.googleapis.com

Click ENABLE.

3 Compartir la carpeta de Drive

La SA necesita permiso de Editor en la carpeta raíz donde vivirá la jerarquía Tipo / Frente / Semana.

3.1 · Abre tu carpeta raíz de Drive

La carpeta que será el "techo" de toda la estructura de protocolos firmados.

3.2 · Compártela con la SA

Click derecho → Share (Compartir).

  • Pega el email de la SA (paso 1.4).
  • Permiso: Editor.
  • Desactiva la casilla "Notify people" — no le mandes mail a un robot 😄

Click Share.

3.3 · Copia el ID de la carpeta

Está en la URL cuando abres la carpeta:

https://drive.google.com/drive/folders/1AbCdEfGhIjKlMnOpQrStUvWxYz1234567

5 Activar Blaze + Budget Alert ($1/mes)

Cloud Functions v2 requiere plan Blaze (pay-as-you-go). En tu caso real costará $0/mes — pero ponemos alertas y caps para garantizar que nunca se cobre por accidente.

💰 Costo realista esperado: $0.00/mes. Tu free tier mensual permite 2 millones de invocaciones de Cloud Functions, 5 GB de Storage y 1 billón de calls de Drive API. Vas a usar <1% de eso.

5.1 · Cambiar a plan Blaze

👉 console.firebase.google.com/project/grapco-demo-2026/usage/details

Click Modify plan → Blaze (Pay as you go) → agrega tarjeta de crédito.

ℹ️ Blaze cobra solo lo que pasa del free tier. Si no pasas (no vas a pasar), no cobra. La tarjeta es una garantía, no un cargo automático.

5.2 · Crear Budget Alert a $1/mes (OBLIGATORIO)

👉 console.cloud.google.com/billing/budgets?project=grapco-demo-2026

Click CREATE BUDGET:

  • Name: GRAPCO Costo Cero
  • Projects: selecciona grapco-demo-2026
  • Services: All services
  • Budget type: Specified amount
  • Target amount: 1.00 USD
  • Threshold rules: 50%, 90%, 100% (vienen por defecto)
  • Email alerts: ✅ marca "Email alerts to billing admins and users"

Click FINISH. Listo — si alguna vez llega a $0.50 te llega un email; si llega a $1 también; si llega a $0.90 también.

🛡️ Cómo funciona: El budget NO apaga nada automáticamente — solo te avisa. Si te llega el primer mail "50% del budget", revisas qué está pasando y decides. En tu caso de uso real, esos mails NUNCA van a llegar.

5.3 · (Opcional avanzado) Cap duro de gasto

Si quieres que cuando llegue a $5 se apague el billing automáticamente (cap duro), Google lo permite via Cloud Function + Pub/Sub:

👉 cloud.google.com/billing/docs/how-to/disable-billing-with-notifications

💡 No es necesario para tu caso. El budget alert a $1 es más que suficiente — si te avisa, puedes desactivar la function manualmente en 30 segundos.

6 Guardar los 3 secrets en Firebase

Abre una terminal en la raíz del proyecto y corre estos 3 comandos. Pega cada valor cuando te lo pida y dale Enter.

6.1 · Posiciónate en la raíz

cd "c:\Users\fjros\OneDrive\Escritorio\PROYECTOS_FRANKLIN2025\PROYECTO GRAPCO 2026\GRAPCO_PROYECTO_COMPLETO"

6.2 · Secret 1 — ID de la carpeta de Drive

firebase functions:secrets:set DRIVE_ROOT_FOLDER_ID --project grapco-demo-2026
# Pega el ID del paso 3.3 y Enter

6.3 · Secret 2 — ID del Sheet de control

firebase functions:secrets:set SHEET_CONTROL_ID --project grapco-demo-2026
# Pega el ID del paso 4.3 y Enter

6.4 · Secret 3 — JSON completo de la Service Account

firebase functions:secrets:set GOOGLE_SA_KEY --project grapco-demo-2026 --data-file="C:\ruta\a\tu\grapco-demo-2026-XXXXX.json"
💡 Si --data-file no te funciona, abre el JSON en Notepad, copia TODO el contenido y usa el comando sin --data-file — te dejará pegarlo.

7 Desplegar la Cloud Function

Un solo comando. Tarda ~2 min.

7.1 · Deploy

firebase deploy --only functions:protocoloPdfFirmadoSync --project grapco-demo-2026

Cuando termine, verás algo como:

✔ functions[protocoloPdfFirmadoSync(us-central1)] Successful create operation.

Llena los campos mínimos (frente, semana, estructura, datos). Guarda.

8.3 · Sube el escaneado (o el mismo PDF para test)

Arrastra el PDF a la zona de drop "Subir PDF firmado". Espera 5-15 segundos.

8.4 · Verifica los 3 destinos

CHECKLIST DE ÉXITO

✓ Firebase Storage en protocolos-firmados/...
✓ Google Drive: aparece en tu carpeta raíz → Pre-Vaciado / F03 / 2026-W22 /

El editor del protocolo también muestra un banner rojo con el mensaje exacto del error si la sincronización falló 3 veces.

9 Troubleshooting común

Los errores más típicos y cómo resolverlos en 30 segundos.

Error Causa y fix
Permission denied al subir a Drive La SA no tiene acceso a la carpeta. Revisa paso 3.2 — el email debe estar como Editor.
API not enabled Falta habilitar Drive o Sheets API. Revisa paso 2.
Unable to parse range: Protocolos!A:N Falta la pestaña Protocolos en el Sheet (paso 4.4) o tiene otro nombre.
Los puntitos verdes no aparecen pero el archivo SÍ está en Drive Delay de Firestore. Refresca la página después de 30s. Si persiste, hay un error en el update() — revisa logs.
Failed después de 3 reintentos Firestore guarda archivado.error con detalles. El banner rojo en el editor te dice qué pasó. Vuelve a arrastrar el PDF para reintentar.
The caller does not have permission en Sheets El Sheet no fue compartido con la SA. Revisa paso 4.2.
