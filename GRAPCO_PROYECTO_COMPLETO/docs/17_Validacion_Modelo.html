📐 El método: Leave-One-Out Cross-Validation

LOOCV es una técnica clásica de machine learning para validar modelos cuando hay pocos datos. Ideal para nuestro caso.

ALGORITMO LOOCV
Para cada Carta Balance N:
predicción = optimizar(actividad_N, datos_excluyendo_N)
error_N = |LUF_real_N − LUF_predicho_N|
Se itera sobre todas las cartas, se promedian los errores → métrica de precisión global

Pasos del algoritmo

  1. Ordenar las Cartas Balance por fecha ascendente — las primeras "entrenan", las últimas "validan".
  2. Para cada carta N a partir de la segunda: tomar todas las cartas anteriores como histórico.
  3. Verificar precondición: que haya al menos 1 carta previa con la misma actividad. Si no, se descarta.
  4. Simular el optimizador: "si solo tuviera los datos hasta N-1, ¿qué cuadrilla hubiera recomendado para esa actividad y tamaño?".
  5. Comparar con la realidad:
    • delta_LUF = |LUF_predicho - LUF_real|
    • error_relativo = (delta_LUF / LUF_real) × 100
    • acierto_personas = % de personas predichas que efectivamente trabajaron
    • sesgo = LUF_predicho - LUF_real (signed)
  6. Agregar métricas globales: promedio, sesgo y desviación estándar.

Métricas de salida

MétricaFórmulaInterpretación
precisionLUF100 - error_relativo_promedio% de acierto del modelo (objetivo: ≥85%)
deltaLufPromediomedia(|delta_LUF|)Error absoluto típico en puntos LUF
aciertoPersonasPromediomedia(% personas coincidentes)Qué tan a menudo recomienda a las personas correctas
sesgomedia(LUF_pred - LUF_real)+ → sobre-estima · − → sub-estima · ≈0 → neutral
desviacionEstandarσ(delta_LUF)Variabilidad del error (estabilidad del modelo)

✅ Validación retrospectiva FUNCIONA
📈 Lectura del resultado:
  • Precisión 92.2% → El modelo es altamente confiable.
  • Δ ±5 pts → Si el modelo predice LUF 60%, el real estará entre 55% y 65%.
  • Sesgo -4.6 → El modelo es conservador, sub-estima ligeramente. Bueno: las cuadrillas reales rinden mejor de lo predicho.
  • Acierto personas 77.8% → ~3 de cada 4 personas recomendadas efectivamente formaron parte de la cuadrilla real.

🎓 Cómo usar esto en la sustentación

Esta vista es perfecta para responder a preguntas técnicas del jurado.

🎯 Pregunta probable del jurado: "¿Cómo sabes que el optimizador realmente funciona y no es solo una heurística cualquiera?"

Tu respuesta sugerida (60-90 segundos)

"Excelente pregunta. Implementé una validación retrospectiva del modelo usando leave-one-out cross-validation, una técnica estándar en machine learning. La idea es: para cada Carta Balance histórica, simulo qué hubiera predicho el modelo si solo tuviera los datos previos a esa medición, y comparo la predicción con el resultado real. Aplicando esta técnica al histórico del proyecto PTARI, el modelo alcanza una precisión del 92% en la predicción de LUF con un error promedio de ±5 puntos. El sistema además detectó un sesgo conservador de -4.6 pts, lo cual es bueno: las cuadrillas reales tienden a rendir levemente mejor que la predicción. Esto está implementado en la pestaña 'Validación' del módulo Carta Balance — pueden verlo en vivo si gustan."
🎬 Tip para la demo en vivo: ten preparado al menos 5-6 Cartas Balance de prueba en tu Firestore antes de la sustentación, para que la pestaña Validación tenga datos para mostrar. Si solo hay 0-2 cartas, sale el "Sin suficientes datos para validar" — menos impresionante.
