🎯 La gran transformación

Hasta el Bloque 20 teníamos módulos potentes pero independientes: producción aquí, materiales allá, calidad por separado. El Bloque 21 los conecta a través de una columna vertebral única: el Plan Maestro WBS. Ahora cada actividad del proyecto tiene su APU, su PET, su protocolo, sus materiales asignados, y todo se cruza automáticamente para producir el Resultado Operativo mensual con proyección al cierre. Es el estándar que usan Cosapi, Graña y Montero, JJC, Aldesa.

📊 Resultado Operativo (RO) — La joya

El reporte mensual que cruza TODO. Como Cosapi, JJC, Graña.

📢 Qué es el RO

El Resultado Operativo es el "Estado de Resultados de la Obra". Cada mes responde 3 preguntas críticas: (1) ¿Cuánto se ha valorizado al cliente este mes? (2) ¿Cuánto cuesta REAL haber producido eso? (3) ¿Cuál es el margen actual y proyectado al cierre? Es el dashboard de un Director de Operaciones de obra.

Estructura del RO Mensual (siguiendo metodología Costos Perú)

1️⃣ Gestión a la fecha

Por cada partida del Plan Maestro:

PartidaAvance % Vendido (PV)Costo AplicadoCosto Real Margen AplicadoMargen Real
02.01.003 Vaciado columnas 49.5% S/. 20,535 S/. 17,455 S/. 18,712 15% 8.9%
02.02.001 Encofrado vigas 72.0% S/. 14,400 S/. 12,240 S/. 11,520 15% 20.0%

2️⃣ Conceptos clave (con su fórmula)

EARNED VALUE (EV) — VALOR GANADO
EV = Avance% × Presupuesto Contractual
Lo que tu trabajo "ha ganado" en el período.
COSTO APLICADO
Costo Aplicado = Avance Real × Costo APU Teórico
Lo que DEBERÍA haber costado producir el avance, según APU original.
COSTO REAL (AC — Actual Cost)
AC = MO real + Materiales reales + Equipos + Subcontratos
Lo que SÍ costó realmente producir el avance.
RESULTADO PENDIENTE
RP = (Costo Aplicado − Costo Real) + Δ Saldo a ejecutar
Análisis combinado: ¿el problema fue el costo o la productividad?

3️⃣ EVM — Indicadores PMI estándar

IndicadorFórmulaInterpretación
CPI (Cost Performance Index)EV / AC>1 = bajo presupuesto · <1 = sobrecosto
SPI (Schedule Performance Index)EV / PV>1 = adelantado · <1 = atrasado
EAC (Estimate at Completion)BAC / CPICosto total proyectado al cierre
ETC (Estimate to Complete)EAC − ACCuánto falta gastar
VAC (Variance at Completion)BAC − EACMargen final proyectado

4️⃣ Proyección al cierre y oportunidades

El sistema identifica automáticamente:

  • Partidas con margen negativo → alerta roja
  • Partidas con sobrecosto en MO > 10% → analizar productividad
  • Partidas con sobrecosto en Mat > 10% → revisar compras
  • Partidas que aportan más al margen → partidas estrella
  • Partidas críticas (sobrecosto + alto peso) → foco de gestión

El flujo de datos automático entre módulos.

Flujo automático de datos (capataz → RO)

// CAPATAZ registra avance en su tableta
Historial.add({ partidaWBS: "02.01.003", metrado: 3.5, fecha: today, hh: 28 });

// → Plan Maestro actualiza avance acumulado de la actividad
PlanMaestro["02.01.003"].avanceMetradoAcum += 3.5;
PlanMaestro["02.01.003"].hhAcumReal += 28;

// → Calidad: ¿hay protocolo abierto para 02.01.003?
if (Protocolos["02.01.003"].estado === "liberado") avanceValido = true;

// → Materiales: ¿se descontaron los materiales segun APU?
consumoTeorico = APUs["02.01.003"].materiales × 3.5;
consumoReal = Kardex.salidas(partida="02.01.003");
deltaMaterial = consumoReal − consumoTeorico;

// → APU real se recalcula
APUs["02.01.003"].apuReal = recalcular(Tareos, Kardex);

// → Valorizaciones contractuales auto se actualizan
ValorizacionContractual.partida["02.01.003"].avancePeriodo += 3.5;

// → RO mensual: nuevos numeros listos para el reporte
RO.mes[actual].partida["02.01.003"].EV = 3.5 × 485.20;
RO.mes[actual].partida["02.01.003"].AC = 28 × salario_promedio + materialesReales;
RO.mes[actual].partida["02.01.003"].CPI = EV / AC;

Todo este flujo ocurre AUTOMÁTICAMENTE. El capataz solo registra avance, el sistema hace la magia.
