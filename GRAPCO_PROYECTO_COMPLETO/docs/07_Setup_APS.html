⚙️ SETUP TÉCNICO · BLOQUE 9

Configurar APS y Cloud Functions paso a paso

Guía completa para activar la subida de modelos Revit a Autodesk Platform Services desde GRAPCO. Toma ~30 minutos seguir todo. Una vez configurado, los usuarios pueden subir .rvt directamente y verlos en el visor 3D.

1 Crear app en Autodesk APS

Necesitas obtener un Client ID y Client Secret.

1.1 Crear cuenta gratuita

Ve a https://aps.autodesk.com y haz click en Sign up. Usa tu email corporativo de GRAPCO. Acepta los términos.

💰 Free tier: 50,000 cargas/mes gratis. Para una obra típica de GRAPCO con 20 usuarios eso son ~2,500 cargas por usuario al mes — MUY suficiente.

1.2 Crear app

Una vez logueado, ve a "Applications" → "Create app":

  • Name: GRAPCO Produc-App
  • Description: Plataforma productividad construcción
  • APIs to enable: Marca Data Management, Model Derivative, OSS, Authentication
  • Callback URL: https://localhost:3000 (no se usa, pero es obligatorio)

Copia el Client ID y Client Secret que te genera. Guárdalos en un lugar seguro.

⚠️ Nunca subas el Client Secret a Git. En este sistema lo configuramos como variable de entorno en Firebase, no queda en código.

2 Habilitar Cloud Functions en Firebase

Las Cloud Functions son el "puente" entre GRAPCO y APS.

2.1 Subir plan de Firebase a Blaze (pay-as-you-go)

Cloud Functions requiere plan Blaze. El plan Blaze incluye un free tier muy generoso:

  • 2 millones de invocaciones/mes gratis
  • 400,000 GB-segundos gratis
  • 5 GB de transferencia/mes gratis

Para GRAPCO esto significa $0/mes en facturación a menos que crezca masivamente.

Ve a Firebase Console → tu proyecto → ⚙️ Settings → Usage and billing → Modify plan → Blaze.

🔒 Configura presupuesto máximo: En Google Cloud Console pon un budget alert de USD 5/mes. Si por algún motivo se dispara el uso, te avisa antes de cobrar.

2.2 Instalar Firebase CLI

# En tu terminal local
npm install -g firebase-tools

# Login con la cuenta del proyecto
firebase login

# En la raíz de tu proyecto GRAPCO
firebase init functions
# Selecciona: tu proyecto Firebase existente
# Lenguaje: JavaScript
# ESLint: No (opcional)
# Instalar dependencias: Yes

2.3 Copiar archivos del backend

Te entrego dos archivos:

  • functions_index.js → copiar a functions/index.js
  • functions_package.json → copiar a functions/package.json

Luego instalar dependencias:

cd functions
npm install

2.4 Configurar credenciales APS como secret

Hay dos formas. La recomendada es usar Firebase Functions config:

# En la raíz del proyecto
firebase functions:config:set \
  aps.client_id="TU_CLIENT_ID_DE_APS" \
  aps.client_secret="TU_CLIENT_SECRET_DE_APS" \
  aps.bucket_key="grapco-models-prod"

# Verificar que se guardó
firebase functions:config:get
📝 Nota sobre bucket_key: debe ser único en TODO Autodesk (es como un dominio). Sugerencia: grapco-models-{nombre-corto-empresa} en minúsculas, sin espacios.

2.5 Desplegar

firebase deploy --only functions

# Output esperado:
✔  functions[apsObtenerUrlSubida(us-central1)] Successful deploy
✔  functions[apsCompletarSubida(us-central1)] Successful deploy
✔  functions[apsConsultarManifest(us-central1)] Successful deploy
✔  functions[apsTokenViewer(us-central1)] Successful deploy
✔  functions[apsEliminarModelo(us-central1)] Successful deploy

Function URL (apsObtenerUrlSubida):
  https://us-central1-tu-proyecto.cloudfunctions.net/apsObtenerUrlSubida

⚠️ Anota la URL base. La necesitas para el siguiente paso.

3 Configurar el frontend

Solo 1 archivo necesita ajuste.

3.1 Editar apsClient.js

Abre src/utils/apsClient.js y verifica las constantes en la cabecera:

const FUNCTIONS_REGION = 'us-central1';
const FUNCTIONS_PROJECT = 'control-productividad-franklin';  // ← TU proyecto Firebase

Si tu proyecto Firebase tiene otro ID, cámbialo. La región normalmente queda en us-central1.

3.2 Copiar archivos nuevos al frontend

Archivo entregadoUbicación destino en tu repo
apsClient.jssrc/utils/apsClient.js
BimUploader.jsxsrc/components/BimUploader.jsx
BimViewerAPS.jsxsrc/components/BimViewerAPS.jsx
BIM.jsxsrc/views/ingeniero/BIM.jsx (reemplazar)

3.3 Reglas de Firestore para nueva colección

En Firebase Console → Firestore → Rules, agrega:

match /BIM_Modelos/{modeloId} {
  allow read: if request.auth != null;
  allow write: if request.auth != null;
}

(Cuando endurezcamos seguridad en general, también restringir esto.)

4 Probar end-to-end

4.1 Test de subida

  1. Abre GRAPCO → login como Ingeniero
  2. Dashboard → grupo 📈 ANÁLISIS → tab 🏗️ Modelo BIM
  3. Click sub-tab 📤 Subir Modelo a APS
  4. Arrastra un archivo .rvt (o usa el botón "click para seleccionar")
  5. Observa la barra de progreso de subida (segundos a 1 min según tamaño)
  6. Cuando llegue al 100% verás "✅ Subida completa. Traduciendo..."
  7. Aparece en la tabla "📦 MODELOS SUBIDOS" con estado 🔄 Traduciendo
  8. Espera 5–30 min (un Revit pequeño tarda 5 min, uno grande 30 min)
  9. Estado pasa a ✅ Listo
  10. Click en 👁️ del modelo → te lleva al visor con el modelo cargado

4.2 Test del visor

  1. Sub-tab 🏗️ Fase 2 · Visor 3D ya tiene el URN cargado
  2. Espera ~10 segundos a que el SDK descargue + renderice
  3. Modelo aparece. Puedes orbitar (mouse), zoom (rueda), pan (shift+mouse)
  4. Click en cualquier elemento del modelo
  5. Aparece panel lateral con el GUID del elemento + propiedades + si está vinculado a GRAPCO
  6. Si NO está vinculado: copia el GUID, ve a Fase 1 · Vinculación y créalo

4.3 Caso de éxito completo

Vinculaste un muro a la actividad "MUROS DE ALBAÑILERÍA". Ahora cuando alguien hace click en ese muro en el visor, ve:

  • ✅ VINCULADO A GRAPCO
  • MUROS DE ALBAÑILERÍA
  • ALBAÑILERÍA · MUROS
  • (En siguientes bloques: avance %, CPI, HH ejecutadas, capataz responsable)

5 Troubleshooting común

SíntomaCausa probableSolución
"Credenciales APS no configuradas"Olvidaste firebase functions:config:setEjecutar paso 2.4 + redespliegue: firebase deploy --only functions
CORS error en consolaURL del backend mal escrita en apsClient.jsVerificar FUNCTIONS_PROJECT coincide con tu Firebase project ID
Subida queda en "Traduciendo" para siempreAPS rechazó el archivo (corrupto o muy grande)Ver logs: firebase functions:log
"Failed to fetch" en frontendPlan Spark, no BlazeSubir a Blaze (paso 2.1) y redesplegar funciones
Visor carga pero sale negroManifest aún no completoEsperar más. Verifica estado en pestaña Subir Modelo
Click en elemento no muestra GUIDModelo no exportó externalIdsEn Revit, revisar que la exportación incluya GUIDs. Reexportar.
